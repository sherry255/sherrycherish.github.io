Linux下删除文件夹的完美解决方案

发表于 2017-02-22 |

　　经常有人用 rm -rf 删除文件夹, 然而,这样做的坏处是:

  • 可能会手抖导致系统清空
  • 命令太长,,,

　　解决方案当然是有的:
　　用 mv /tmp

　　/tmp - 临时(temporary)文件, 会在重启后清空

　　这么做也有问题, 因为服务器是24小时开启的, 这样的话 /tmp 就会堆积大量文件.

解决问题的思路如下:

  • 新建一个 /ttt 文件夹
  • 将其设置为: 1, 重启后清空 2, 利用crontab设置一个定时任务, 定时清空该文件夹.
    　　http://linuxtools-rst.readthedocs.io/zh_CN/latest/tool/crontab.html

Linux 和 Unix 文件系统

发表于 2017-02-21 |

Linux 和 Unix 文件系统被组织成一个有层次的树形结构。文件系统的最上层是 /，或称为 根目录。在 Unix 和 Linux 的设计理念中，一切皆为文件——包括硬盘、分区和可插拔介质。这就意味着所有其它文件和目录（包括其它硬盘和分区）都位于根目录中。 例如：/home/jebediah/cheeses.odt 给出了正确的完整路径，它指向 cheeses.odt 文件，而该文件位于 jebediah 目录下，该目录又位于 home 目录，最后，home 目录又位于根(/) 目录下。 在根 (/) 目录下，有一组重要的系统目录，在大部分 Linux 发行版里都通用。直接位于根 (/) 目录下的常见目录列表如下：

  • /bin - 重要的二进制 (binary) 应用程序
  • /boot - 启动 (boot) 配置文件
  • /dev - 设备 (device) 文件
  • /etc - 配置文件、启动脚本等 (etc)
  • /home - 本地用户主 (home) 目录
  • /lib - 系统库 (libraries) 文件
  • /lost+found - 在根 (/) 目录下提供一个遗失+查找(lost+found) 系统
  • /media - 挂载可移动介质 (media)，诸如 CD、数码相机等
  • /mnt - 挂载 (mounted) 文件系统
  • /opt - 提供一个供可选的 (optional) 应用程序安装目录
  • /proc - 特殊的动态目录，用以维护系统信息和状态，包括当前运行中进程 (processes) 信息。
  • /root - root (root) 用户主文件夹，读作“slash-root”
  • /sbin - 重要的系统二进制 (system binaries) 文件
  • /sys - 系统 (system) 文件
  • /tmp - 临时(temporary)文件
  • /usr - 包含绝大部分所有用户(users)都能访问的应用程序和文件
  • /var - 经常变化的(variable)文件，诸如日志或数据库等

电视剧<硅谷>看后感

发表于 2017-01-11 |

我平时是不看电视剧更不看任何娱乐节目的, 有段时间, 感觉自己压力太大= =, 一直在看书、学习、写代码， 不敢怠慢一点。

于是乎效率是真的低了好多。。。我痛定思痛， 决定娱乐一番。于是就找了部最适合程序员看的电视剧：《硅谷》，以下是总结：

创业公司可能会遇到的坑：

  • 被投资人骗去技术：电视剧中的压缩技术被盗用= =
  • 关于空降的CEO： 非技术型导向的公司的话, CEO不懂技术的话, 其实可能也没关系. 但是对于技术导向型的公司来说, CEO完全不懂技术的话, 就有可能判断错误.

关于geek:

  • 影视剧是非常喜欢把geek形象夸张化的,大致是这样的形象: 类似IT狂人中那样: 不善言辞, 邋遢,不懂生活, 一心泡在代码中.

  • 事实上, 会生活的人, 才会工作.

  • 生活质量、健康状况、身体素质等决定了一个人精力是否旺盛，是否能够有足够的精力做很多事情。
